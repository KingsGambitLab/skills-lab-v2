## NEW friction (v4-found)

### P1 — `_next_action`'s web branch builds non-precise dashboard URL

**Where:** `cli/src/skillslab/cli.py:498-510` (web branch of `_next_action`). Builds URL from `course_id` alone:

```python
url = f"{state.web_url().rstrip('/')}/#{course_id}"  # line 506
```

The cursor entry has `module_id` and `step_pos`; we see them flowing correctly through `_print_web_pointer` on the same step via `status` (3-part URL there). There are TWO URL-emission sites: line 506 here, and line 653 in `_print_web_pointer`. Commit 1785682 fixed `_browser_url` and migrated `_print_web_pointer` to call it; `_next_action` was missed.

**Why P1:** This is exactly the live-miss class v3 caught and the shim was built to encode. Invariant II audited the FUNCTION; it didn't audit call sites. After `skillslab next` advances onto a WEB step, CTA URL drops module/step precision; browser hash-router restores from stale cursor; learner lands on wrong step.

**Fix (1-line):** replace line 506 with `url = _browser_url(course_id, step)`. Same shape, correct builder.

### P2 — Bottom CTA panel after `spec` advertises "Read briefing"

After `skillslab spec` renders, the green CTA panel says PRIMARY = "Read briefing / $ skillslab spec" and "then: skillslab check — Run + grade". Learner just DID spec; the primary should be `check`. `_next_action` doesn't know caller context. Fix: thread `just_did="spec"` into `_next_action` and drop the spec entry from the primary slot when the caller IS `spec`. Or `spec` calls a variant `_next_action_after_spec`.

### P3 — Concept-widget HTML degrades to fragmented prose (carryover)

Same as v3 P3 (a). Kimi M0.S1's `2.5x` headline + widget content arrives as fragmented prose. Workable. Backlog.

## Suggestions for v5+ harness expansion

1. **Tighten Invariant II to audit ALL URL-emission sites.** Grep cli.py for every `f"{...web_url()...}/#{...}"` pattern, assert every match line is INSIDE the body of `_browser_url`. Would have caught the new P1.
2. **Add a `_next_action`-shape invariant.** For each (surface × exercise_type), assert PRIMARY action label != the calling command's name. Would have caught P2 (`spec` returning "Read briefing / skillslab spec" is a self-loop).
3. **Render-snapshot test.** Pin step 85139 + 85059 + 85152 + 85164 + 85086; snapshot `step_to_markdown` output; fail on diff.
4. **Tighten Invariant I's regex** to require CTA-shape context (link markup / panel content / Markdown code-fence delimiters), eliminating the 3 false-positives in v4 (capstone / config / image).
5. **Synthetic `_next_action` driver.** Pure-function unit test: drive every branch with fake step dicts; assert URL is 3-part for web, primary action is `check` for terminal+cli_commands, etc.

Bottom line: the shim is paying for itself — v4 took ~25 min vs v3's ~1.5 hr AND found the live-miss-class the shim missed. The new P1 reveals a generalisation: move from "is this function correct" to "is every site that emits URLs going through this function." That's the right invariant shape.
