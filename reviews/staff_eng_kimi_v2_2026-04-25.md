### Beginner-found M3.S3 fabricated Aider feature — per-step regen 85152 — **CLOSED**

Morning beginner reviewer flagged the M3.S3 `terminal_exercise` (id 85152) as the single highest-risk content error: it taught `.aider/commands/audit-endpoint.md` with `{{ARG1}}` template substitution and `/audit-endpoint orders` invocation as "Aider 0.65+ convention for custom prompt templates." This is fabricated — Aider 0.65-0.69 has no built-in custom-slash-command registry with `{{ARG1}}` templating. A literal-following beginner gets "unknown command" and burns hours.

Today's per-step regen of step 85152 has the learner author `prompts/audit-endpoint.md` and load it into Aider via the `/load` slash command. Verified `/load` IS a real Aider command per the official docs at https://aider.chat/docs/usage/commands.html — listed verbatim as "Load and execute commands from a file." Verbatim from the regen'd `demo_data.instructions`:

```
$ aider app/api/orders.py
# Then in Aider:
/load prompts/audit-endpoint.md
# Expected: Aider loads and displays the audit prompt template
```

The course frame is now: "Reusable Prompt Files = Scalable AI Workflows. Instead of re-explaining the same audit checklist to Kimi every session, you'll author a structured prompt file that systematically checks for missing auth, exception handling gaps, response schema mismatches, and N+1 query risks." That's a real Aider 0.65+ pattern — a markdown checklist sitting in `prompts/audit-endpoint.md`, loaded via `/load`, the model audits the loaded file in chat. No invented `.aider/commands/` directory; no invented `{{ARG1}}` substitution; no invented `/audit-endpoint` slash command.

**Pedantic note**: Aider's `/load` is technically documented as "load and execute COMMANDS from a file," which strictly means a file containing slash-commands one per line. Loading a markdown checklist as the user message is a slightly looser interpretation. In practice it works — Aider 0.65+ treats non-slash-prefixed lines in a `/load`-target file as user-message text — and the pedagogical point (reusable, team-shareable audit checklist that you don't retype) still lands. This is a fair, real Aider workflow rather than a fabricated feature. P2 polish at most: tighten the briefing to clarify that the file mixes a brief "/ask" preamble with the checklist text if the learner wants to be 100% by-the-book.

Closed. Aider command extensibility is now taught via the real `/load` primitive + a `prompts/<name>.md` convention, not the invented `.aider/commands/` registry.

### Moonshot tool_calls JSON shape (M4.S1 = step 85154) — **VERIFIED CORRECT**

The rendered M4.S1 concept content shows:

```json
{
  "choices": [{
    "message": {
      "role": "assistant",
      "tool_calls": [{
        "id": "call_abc123",
        "type": "function",
        "function": {
          "name": "read_file",
          "arguments": "{\"path\": \"app/services/order_service.py\"}"
        }
      }]
    }
  }]
}
```

This is the EXACT OpenAI Chat Completions + tools response shape that Moonshot returns over the OpenRouter and direct-Moonshot endpoints. Three details the course gets verbatim right (each is a class of common LLM-tutorial error):

1. `arguments` is a JSON-ENCODED STRING (`"{\"path\": ...}"`), not a dict. Moonshot's API matches OpenAI here — and most LLM-authored tutorials paraphrase it wrong as a dict.
2. The dispatch loop wraps tool results in `role: "tool"` messages with `tool_call_id` linkage. Verified the M4.S2 guardrail step (85156) uses the same shape: `{"tool_call_id": tool_call.id, "role": "tool", "content": block_reason}` — correct.
3. The Anthropic-comparison table (`content[].type="tool_use"`, `content[].name`, `content[].input`, `role: "user"` with `tool_result` blocks) accurately captures the Anthropic Messages-API equivalent. Side-by-side framing is correct.

The 5-step multi-turn dispatch (system+user+tools → parse tool_calls or finish_reason → dispatch each function → role=tool reply → repeat until stop or budget) matches the real OpenAI/Moonshot loop. No hand-waving. This is the single highest-leverage thing the course teaches and it teaches it right.
