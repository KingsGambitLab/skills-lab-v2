## NEW friction points (by severity)

**P1 — `validation.cli_commands` not rendered in the briefing.** Every regenerated `terminal_exercise` step has `cli_commands` in DB; `check.py` dispatch runs them; runner shows them per-cmd live. But `step_to_markdown` (render.py:333-370, the "Acceptance" block) has branches for `cli_check` (legacy) and "rubric only" — no branch for `cli_commands`. Learner reading `skillslab spec` cannot see what `skillslab check` will actually do. **Fix:** ~30 lines added to render.py:333 — see G4 above for the rendered shape. **Why P1:** breaks the headline v3 promise. The discoverability gap pushes beginners toward the wrong mental model on their first terminal_exercise.

**P2 — `--course` help text still says `aie`.** cli.py:127 has `help="Course slug (kimi / aie / jspring) — ..."`. The only learner-facing surface that still says `aie` post-rename. **Fix:** change to `kimi / claude-code / jspring`. One-line edit.

**P2 — `--paste <run-url>` advertised but doesn't feed the GHA runner.** cli_runners.py:494's GHA-watch timeout error tells the learner: "Re-run `skillslab check` once it's done, or pass `--paste <run-url>`." But the check command's `--paste` option (cli.py:982) feeds legacy `_build_submission`, not the GHA runner — pasting a run URL goes through bridge_validate as a generic submission, which may or may not work depending on the LMS-side validator. **Fix:** either make `--paste <url>` short-circuit and submit URL directly (simpler), or drop the `--paste` line from the timeout message.

**P3 — Concept-widget HTML degrades to fragmented prose.** Kimi M0.S1 + claude-code M0.S1 are concept widgets with embedded JS. CLI strips scripts/styles correctly but leaves prose disconnected from its layout containers → wall of single-line fragments. **Fix:** when a stripped widget produces >5 consecutive single-line fragments, collapse to `[interactive widget — see browser]`. Or aggressively: for `concept + surface=web`, skip the body render entirely and just emit the WEB-step pointer panel (the CLI already does this for some cases — extend to all).

**P3 — Stale step files after a regen.** Pre-regen cached files in `~/.skillslab/kimi/steps/` showed `kimi-k2` (no `-0905`) until I re-ran `skillslab start kimi`. `start` is idempotent (cli.py:308 deletes old before writing) but no signal to the learner their files are stale. **Fix:** persist a `course_content_etag` on `start`, compare on `spec`/`status`/`next`, emit "⚠ course content updated; run `skillslab start <slug>` to refresh."
