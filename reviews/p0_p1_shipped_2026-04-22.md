## Remaining gaps (what the stress-test exposed)

### Gap A — LLM still tries to write complete starters

Even with the tightened prompt ("STARTER MUST NOT PASS"), the LLM sometimes emits a working solution in the `code` slot and a duplicate of it in `solution_code`. The LangGraph gate catches this, but each rejection burns one Docker roundtrip + one LLM call (~$0.05 + 15-25s). 5 retries on one course added ~$0.25 and 120s to generation time.

**Mitigation ideas**:
- Add a pre-LangGraph check: quickly grep the starter for `pass`, `# TODO`, `raise NotImplementedError`, or known wrong-sentinel returns. If none present, reject before wasting a Docker check.
- Add few-shot examples to the Creator prompt showing the correct broken-starter pattern.
- Consider using structured output mode for the two fields so the LLM commits to "starter = scaffold, solution = complete."

### Gap B — 2/7 code_exercise steps fell back to legacy `must_contain`

When the retry budget is exhausted, the Creator ships the legacy path. That's safe (generation doesn't block) but it means those specific steps can still be cheesed. A learner writing `print("return []")` would satisfy the `must_contain` substring for `return []` in the legacy path.

**Mitigation**: bump retry budget from 3 to 5 for code_exercise + add a "please emit broken starter" system prompt on the final retry. Alternatively, fail-closed (reject the whole step) once the LangGraph path has been attempted.

### Gap C — Docker slower than mock sandbox

Per-test Docker run ≈ 3-12s (first pytest install dominates). Legacy sandbox ≈ 0.1-0.5s. For a learner submitting 5 code_exercises in a session, the time difference is 15-60s vs 1-3s. Acceptable for real tests + no-mocks but worth monitoring.

**Mitigation**: ship a pre-built `sll-python-runner:latest` image with pytest + common libs pre-installed. Takes per-run down to ~1-2s (just the test execution). Would do this next if this stays a bottleneck.

### Gap D — Remote Docker + prewarming

Remote box's `/var/run/docker.sock` is accessible by ec2-user (systemd `SupplementaryGroups=docker` added this session). But Docker images haven't been prewarmed on the remote — first real-grading call on 18.236.242.248 will pull python:3.11-slim for 20-30s. Should run `docker pull python:3.11-slim && docker pull node:20-slim && docker pull golang:1.22-alpine` on the remote before any learner hits it.

### Gap E — Frontend templates shipped but not wired

`frontend/templates/*` exists and is served. But `frontend/index.html` still uses its old inline-render path for every exercise type. The template loader + per-type mount call needs a follow-up PR to replace the existing render functions. Low-risk because the new templates are additive; the legacy path still works.
