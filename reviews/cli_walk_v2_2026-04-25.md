## 4. NEW friction points

### P2-A — Post-`check`-pass "next" hint is flat dim text, not a Panel

When `skillslab check` passes a step, `cli.py:959-960` prints:
```python
console.print(f"\n[bold]Next:[/bold] M{nxt['module_pos']-1}.S{nxt['step_pos']} — {nxt['title']}")
console.print("[dim]skillslab spec[/dim]")
```
That's the exact `[dim]flat text[/dim]` shape v1 caught for `status` (P1-5). The v2 fix-round upgraded `status`'s CTA to `_print_next_actions(_next_action(...))` but didn't apply the same upgrade to this branch.

Concrete fix sketch:
```python
# replace the two console.print calls with:
actions = _next_action(slug, meta, nxt)
_print_next_actions(actions)
```
Same one-liner that fixed `status`. Apply at `cli.py:958-960` (inside the `if cur + 1 < len(...)` branch).

Severity: **P2.** Reaches the user only after a successful `check` pass, which is a rarer surface than the daily-driven `status` / `next` / `spec` commands. The flat-text version still tells the learner what to do; it just doesn't have the same "this is the action" visual emphasis. Doesn't block ship — but it should land in the next CLI-walk pass, alongside the paragraph-break-loss content fix and the slug-coverage cleanup, as a "we said all next-action CTAs are Panels now, finish the job" change.

### P2-B — `must_contain` description fallback for non-pattern tokens

For tokens like `aider`, `Python 3.1`, `Kimi`, the `_describe_must_contain_token` helper falls through to its generic fallback `"evidence string (must appear in your paste)"`. Functional, but a beginner reading three bullets all suffixed `— evidence string (must appear in your paste)` learns nothing about why each one matters. Pattern-matching for tool-name tokens (`aider`/`gh`/`docker`/`kubectl`/`pytest` etc.) and language-version tokens (`Python 3\.\d+`, `Java \d+`, `Node \d+`) would close most of the gap. **Pre-existing; not a v2 regression.** P2 + low-effort fix.

### Nothing P0 or P1 introduced by v2

Sections A through G found no new P0 or P1 issues. Every `host.docker.internal` test, every `[dim]…[/dim]` flat-text grep, every empty-section check passed for the surfaces v2 claimed to fix.
